1. WHAT IS HEALTHSEC?
================================================================================

HEALTHSEC is a system designed for hospitals and healthcare facilities to:

• Track and monitor security risks in their computer systems
• Check if they're following healthcare regulations (HIPAA, NDPR, ISO 27001)
• Keep a record of who accessed patient data and when
• Generate compliance reports for regulatory audits
• Alert staff when security threats are detected
• Maintain an unchangeable audit trail for legal purposes

Think of it like a "Security Guard" for a hospital's digital systems that:
- Watches for suspicious activity
- Makes sure rules are being followed
- Keeps detailed records of everything that happens
- Alerts management when there's a problem

3.3 MONITORING - Suspicious Activity
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

WHAT: Flags unusual access to patient records
WHY: Detect if someone is accessing data they shouldn't access
WHAT GETS FLAGGED:
  • Access at 2am (hospitals work 9-5, so 2am is suspicious)
  • Accessing many records in a short time (e.g., 100 records in 1 hour)
  • Accessing records outside your department
  • Accessing critical patient records
  • Using a new/unknown device

HOW TO USE:
  1. Click "Monitoring" → "Suspicious Activity"
  2. See all flagged suspicious accesses
  3. If you see something suspicious, click it
  4. Investigate: Was this legitimate? (e.g., Dr. Smith had to work late?)
  5. Click "Resolve" to mark it as checked

REAL WORLD EXAMPLE:
  "Janitor accesses 50 patient records at 3am from a new device.
   System flags this as suspicious because:
   - Janitor isn't a clinician
   - 3am is outside work hours
   - Accessing way too many records

   Manager investigates: Janitor's account was hacked!
   Clicks 'Resolve' to mark the investigation done,
   Changes janitor's password."

WHY: Measure how vulnerable you are to cyber attacks
WHAT YOU SEE:
  • Overall Risk Score (0-100)
    - 0-40 = Low risk (good)
    - 40-70 = Medium risk (getting risky)
    - 70-100 = High risk (very dangerous)
  • Number of open threats
  • Number of unpatched vulnerabilities (security fixes not applied)
  • Charts showing threat trends
  • List of recent threats

HOW TO USE:
  1. Click "Risk Engine" → "Risk Dashboard"
  2. Look at the big score - is it red (bad) or green (good)?
  3. If score is high:
     - Check "Open Threats" - what's wrong?
     - Check "Unpatched Vulns" - what needs to be fixed?
  4. Scroll down to see recent threats

REAL WORLD EXAMPLE:
  "Risk score is 72 (CRITICAL - red).
   Reasons:
   - 5 open threats (ransomware attempts)
   - 12 unpatched vulnerabilities (Windows updates not installed)

   Hospital IT manager:
   1. Patches all Windows updates
   2. Activates antivirus scanning
   3. Checks the threats

   Score drops to 45 (medium) - much better!"

WHAT: Shows if you're following healthcare regulations
WHY: Hospitals MUST follow HIPAA, NDPR, ISO 27001 rules or face fines/lawsuits
WHAT YOU SEE:
  • Overall Compliance Score (0-100%)
    - 0-40% = Not compliant (in violation!)
    - 40-70% = Partially compliant (need to fix)
    - 70-100% = Compliant (good!)
  • Per-framework scores (HIPAA, NDPR, ISO 27001)
  • Count of controls (rules):
    - Passing (you're following them)
    - Failing (you're not following them)
    - Partial (you're doing some but not all)

HOW TO USE:
  1. Click "Compliance" → "Dashboard"
  2. Look at overall score - aim for 90%+ to be safe
  3. If score is low, click "Run Automated Check"
  4. System tests all compliance rules and reports results
  5. Go to "Remediation" to fix failing controls

REAL WORLD EXAMPLE:
  "Hospital compliance score: 75% (mostly compliant)

   Breakdown:
   - HIPAA: 82% (good)
   - NDPR: 71% (needs work)
   - ISO 27001: 73% (needs work)

   Hospital:
   - Runs compliance check
   - Sees 3 NDPR controls failing (password policies)
   - Updates password rules to meet NDPR requirements
   - Score improves to 85%"

   - May 26, 2pm: Cleaned infected computers

   Recovery Phase:
   - May 26, 5pm: Restored from backup

5. KEY FEATURES EXPLAINED
================================================================================

FEATURE 1: AUTOMATIC THREAT DETECTION
─────────────────────────────────────

The system automatically analyzes these threat indicators:
  • Brute force attacks (many failed login attempts)
  • SQL injection attempts (hacker code in login fields)
  • Phishing campaigns (suspicious emails)
  • Privilege escalation (user tries to get admin access)
  • Data exfiltration (unusual large downloads)
  • Malware indicators (known malicious files/IPs)

What you do:
  - System detects threat
  - System creates alert
  - You investigate
  - You take action (block IP, patch, etc.)
  - You mark as resolved


FEATURE 2: COMPLIANCE AUTOMATION
────────────────────────────────

The system automatically tests compliance with:
  • HIPAA (US healthcare privacy law)
  • NDPR (Nigeria data protection law)
  • ISO 27001 (International security standard)

Tests include:
  ✓ Is encryption enabled?
  ✓ Are passwords strong enough?
  ✓ Are audit logs being kept?
  ✓ Are backups being tested?
  ✓ Is access control working?
  ✓ Are security patches applied?

You don't have to manually test everything - the system does it for you.


FEATURE 3: ROLE-BASED ACCESS
────────────────────────────

Different users see different things:

  VIEWER (read-only)
  - Can see dashboards
  - Cannot make changes
  - Good for management, board

  ANALYST
  - Can see everything viewer sees
  - Can acknowledge/resolve alerts
  - Can mark suspicious activity as reviewed
  - Can patch vulnerabilities

  COMPLIANCE OFFICER
  - All analyst permissions
  - Plus: Run compliance checks
  - Plus: Fix compliance controls
  - Plus: Access audit logs
  - Good for: Compliance manager

  ADMIN
  - Everything
  - Manage users
  - System settings
  - Good for: IT director


FEATURE 4: IMMUTABLE AUDIT LOG
──────────────────────────────

The audit log is "write-once, never erase":
  • Every action is recorded
  • Records are cryptographically signed
  • Cannot be modified or deleted (even by admins)
  • Tamper-evident - any modification is detected
  • Legal proof that you're following rules

Why this matters:
  - Regulators trust the log because it can't be faked
  - In a lawsuit, you have proof of what happened
  - If hacked, the audit log shows how they got in


FEATURE 5: RISK SCORING
──────────────────────

Risk score is calculated from:
  • Number of open threats
  • Number of unpatched vulnerabilities
  • Missing security controls
  • Outdated systems
  • Weak passwords
  • Missing encryption
  • No backup/recovery plan

Score ranges:
  0-40 = Low risk (green) ✓
  40-70 = Medium risk (yellow) ⚠
  70-100 = High risk (red) ✗

The score helps you prioritize:
  - High risk? Fix vulnerabilities and threats
  - Medium risk? Implement missing controls
  - Low risk? Maintain current practices

7. TROUBLESHOOTING
================================================================================

PROBLEM: I can't log in
SOLUTION:
  • Check username spelling
  • Check password is correct
  • Check if account is active (not disabled)
  • Try resetting password
  • Contact IT administrator

PROBLEM: My score is going up (getting worse)
REASON: This usually means:
  • New threats were detected
  • Vulnerabilities weren't patched
  • Controls failed automated testing

ACTION:
  • Check "Recent Threats" - what's new?
  • Check "Vulnerabilities" - which need patching?
  • Run compliance check - which controls are failing?
  • Fix the issues
  • Score will improve as you fix things

PROBLEM: Compliance check is running but shows no results
REASON: Check is still in progress
SOLUTION:
  • Wait 5-10 minutes
  • Refresh the page
  • Check back later

PROBLEM: I can't see a page I need
REASON: Might be permission issue or page doesn't exist
SOLUTION:
  • Check your user role - do you have permission?
  • Ask administrator to grant permission
  • Check if page might be archived/hidden

PROBLEM: Alert won't resolve
REASON: System might be re-detecting the same issue
SOLUTION:
  • Make sure you actually fixed the problem
  • Run a new risk assessment
  • If problem still exists, system will create new alert
  • Fix the underlying issue, not just the alert

PROBLEM: Audit log is empty
REASON: This is unusual - should always have records
SOLUTION:
  • Check if audit logging is enabled
  • Check if you have permission to view audit log
  • Contact IT administrator

8. TECHNICAL DETAILS FOR USERS
================================================================================

WHAT IS HIPAA?
──────────────
Health Insurance Portability and Accountability Act (US law)
Requires:
  • Patient data must be encrypted
  • Access must be logged
  • Passwords must be strong
  • Security patches must be applied
  • Backups must be tested
  • Breach notification in 30 days
Violation fine: $100 - $50,000 per offense

WHAT IS NDPR?
─────────────
Nigeria Data Protection Regulation (Nigerian law)
Requires:
  • Personal data protection
  • Consent for data collection
  • Data subject rights
  • Breach notification
  • Data security measures
Similar to HIPAA but for Nigeria


WHAT IS ISO 27001?
───────────────────
International standard for information security
Requires:
  • Risk assessment
  • Access control
  • Encryption
  • Incident response
  • Business continuity
  • Regular audits
Used globally for certification


WHAT IS PHI?
─────────────
Protected Health Information - any patient data:
  • Names
  • Medical record numbers
  • Birth dates
  • Test results
  • Diagnoses
  • Medications

Must be:
  • Encrypted
  • Logged when accessed
  • Restricted to who needs it
  • Backed up regularly


WHAT IS A CVE?
───────────────
Common Vulnerabilities and Exposures
A database of known security weaknesses:
  • CVE-2024-5678 = a specific known vulnerability
  • CVSS 9.8 = severity score (0-10, higher = worse)
  • Usually has a patch/fix available


HOW DOES ENCRYPTION WORK?
──────────────────────────
Non-encrypted (dangerous):
  Patient data stored as plain text
  If hard drive stolen: data is readable

Encrypted:
  Patient data scrambled with a key
  If hard drive stolen: data is unreadable garbage
  Only person with key can decrypt

HEALTHSEC checks: "Is encryption enabled?"


WHAT IS 2-FACTOR AUTHENTICATION (2FA)?
───────────────────────────────────────
Normal: Username + Password
  Problem: If password stolen, account is compromised

2FA: Username + Password + Something else
  Something else could be:
    • Code from your phone
    • Fingerprint
    • Face recognition
    • USB key

Hackers need both password AND 2FA to get in
Much more secure than password alone


HOW DOES BACKUP AND RECOVERY WORK?
───────────────────────────────────
Daily: System backs up patient data to another location
Monthly: Hospital tests if backup works (restore a test file)
  Why test? Because untested backups often fail when you need them

HEALTHSEC checks:
  ✓ Is backup happening daily?
  ✓ Is backup being tested monthly?
  ✓ Can we restore files quickly?


SECURITY INCIDENT PHASES
─────────────────────────
Detection: "We found a problem"
Containment: "We stopped it from spreading"
Eradication: "We removed the problem"
Recovery: "We brought systems back online"
Post-Incident: "We documented what happened"
